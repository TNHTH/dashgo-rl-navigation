## 新增Skills详解

### 1. Kaizen - 持续改进 ⭐⭐⭐⭐⭐

**何时使用**:
- 编写或重构代码时
- 设计系统架构时
- 改进工作流程时
- 处理错误和验证时

**核心价值**:
- 应用持续改进理念
- 避免过度工程化
- 防错设计(Poka-Yoke)
- 遵循既定模式

**四大支柱**:

1. **持续改进**: 小改进持续进行，而非一次性大改
2. **防错设计**: 在设计阶段防止错误，而非事后修复
3. **标准化工作**: 遵循已验证的模式
4. **及时生产**: 只构建当前需要的，不预判未来

**使用示例**:
```
你: "我想优化这个函数"

Claude (应用kaizen):
我会建议小改进而非重写:
1. 先让它工作
2. 再让它清晰
3. 最后让它健壮
每个步骤都是完整的、可测试的。
```

**实际效果**:
- 避免大规模重构的风险
- 每次改进都是可验证的
- 代码质量稳步提升

### 2. Brainstorming - 头脑风暴 ⭐⭐⭐⭐⭐

**何时使用**:
- 创建新功能前
- 构建组件前
- 添加功能或修改行为前
- 任何创意工作开始前

**核心价值**:
- 通过结构化提问将粗略想法转化为完整设计
- 避免返工
- 探索多种方案

**关键原则**:
- 一次只问一个问题
- 优先使用多选题
- 提出2-3种方案并说明权衡
- 分段展示设计(200-300字)，逐步验证

**使用示例**:
```
你: "我想添加一个用户认证功能"

Claude (应用brainstorming):
Q1: 这个功能的主要目标是什么？
A: [你回答]

Q2 (多选): 应该支持哪些认证方式？
   a) 邮箱+密码
   b) OAuth (Google, GitHub)
   c) 手机号验证
   d) 其他

[继续探索...]

设计验证:
第一部分: 架构设计 (200-300字)
[展示设计]
这样对吗？需要调整吗？

[继续下...]
```

**实际效果**:
- 需求理解更清晰
- 设计考虑更全面
- 减少后期返工

### 3. File Organizer - 文件组织器 ⭐⭐⭐⭐⭐

**何时使用**:
- Downloads文件夹混乱
- 找不到文件
- 有重复文件占用空间
- 文件夹结构不合理
- 需要建立更好的组织习惯

**核心价值**:
- 自动执行你的文件组织规则
- 查找重复文件
- 建议更好的组织结构
- 减少认知负担

**遵循你的规则**:
- **非项目文档** → `D:\cursor\file\Si Yuan\claude\`
- **项目文档** → 项目目录
- **临时文件** → `D:\cursor\file\.claude-temp\`

**文件命名规范**:
- 中文文件名（便于识别）
- 格式: `主题_类型_YYYY-MM-DD.md`
- 或: `主题_说明.md`

**使用示例**:
```
你: "我的Downloads文件夹有500多个文件，帮我整理"

Claude (应用file-organizer):
1. 分析文件类型和数量
2. 建议组织结构:
   - 项目文件 → 各自项目目录
   - 非项目文档 → Si Yuan\claude\
   - 临时文件 → .claude-temp\

3. 展示计划并请求确认

4. 执行整理

整理完成！创建了X个文件夹，移动了Y个文件...
```

**实际效果**:
- 文件井井有条
- 快速找到所需文件
- 定期维护提示

### 4. Tapestry - 内容提取与行动规划 ⭐⭐⭐⭐⭐

**何时使用**:
- 从URL学习内容并创建行动计划
- 提取文章、视频、PDF内容
- 将学习转化为行动
- 避免只学习不实践

**核心价值**:
- 自动检测内容类型（YouTube/文章/PDF）
- 提取干净内容
- 自动创建Ship-Learn-Next行动计划
- 强调"行动优先"

**支持的内容类型**:
1. **YouTube视频**: 提取字幕
2. **文章/博客**: 提取正文
3. **PDF文档**: 提取文本

**触发关键词**:
- tapestry, weave, plan, actionable
- extract and plan, make a plan, turn into action

**使用示例**:
```
你: "tapestry https://www.youtube.com/watch?v=xxxxx"

Claude (应用tapestry):
🧵 Tapestry工作流启动...
📍 检测到: YouTube视频
📺 提取字幕...
✓ 保存到: Si Yuan\claude\extracted-content\视频标题.txt

🚀 创建行动计划...
✓ 保存到: Si Yuan\claude\plans\Ship-Learn-Next Plan - 学习主题.md

✅ 完成！
🎯 你的任务: [一句话总结]
📍 本周目标: [Rep 1]

你什么时候能完成Rep 1？
```

**实际效果**:
- 学习内容被提取并保存
- 自动创建可执行的计划
- 防止"收藏但不学习"

**保存位置**:
- 内容: `D:\cursor\file\Si Yuan\claude\extracted-content\`
- 计划: `D:\cursor\file\Si Yuan\claude\plans\`


**示例对比**:

<Good>
```python
# 迭代1: 让它工作
@mcp.tool("get_user")
async def get_user(user_id: str):

# 迭代2: 让它清晰（添加验证）
# 迭代3: 让它健壮（添加错误处理）
```
</Good>

<Bad>
```python
# 一次性过度设计
@mcp.tool("get_user")
async def get_user(
    user_id: str,
    include_cache: bool = True,
    cache_ttl: int = 300,
    retry_count: int = 3,
    # ... 许多"以防万一"的参数
):
```

</Bad>

## 常见问题

### Q1: Skills会自动触发吗？

**A**: 是的！Claude会根据你的描述自动选择最相关的skill。你也可以明确指定：
```
你: "使用brainstorming帮我设计这个功能"
```

### Q2: Skills可以同时使用吗？

**A**: 可以的！多个skills可以协同工作：
```
你: "使用brainstorming和kaizen设计新功能"

Claude会同时应用:
- brainstorming: 结构化提问
- kaizen: 避免过度设计
```

### Q3: 文件保存位置可以修改吗？

**A**: 可以！file-organizer已经整合了你的文件组织规则。如果你想修改：
```
你: "把文档保存到D:\my-docs\而不是Si Yuan\claude\"
```

### Q4: Tapestry支持哪些网站？

**A**: 当前支持：
- ✅ YouTube (提取字幕)
- ✅ 大部分文章/博客 (提取正文)
- ✅ PDF (提取文本)

如果某个网站不支持：
```
你: "tapestry https://example.com"

Claude: "这个网站需要登录，无法提取。试试其他工具？"
```

### Q5: Kaizen会让我的代码变慢吗？

**A**: 不会！Kaizen强调：
- 先让它工作（简单实现）
- 再让它清晰（重构）
- 最后让它健壮（优化）

只有测量后才优化（"优化前先分析"）

### Q6: File Organizer会删除文件吗？

**A**: 永远不会！除非你明确确认：
```
Claude: "发现23组重复文件（156MB）
      建议：保留最新的，删除其他？

      删除这些文件？(yes/no)"
```

### Q7: Skills使用Windows命令吗？

**A**: 是的！所有skills都已优化为Windows兼容：
- PowerShell命令
- Windows路径（`\`）
- UTF-8编码
- Windows特有错误处理
