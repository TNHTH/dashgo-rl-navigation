### 峰会核心宣言

峰会最重要的宣告是：**2026年将从单纯的"大模型竞争"转向"智能体（Agent）生态"的合力建设**。

### 三位一体转型

```
Chat（聊天）→ Thinking（思考）→ Coding（编程）→ Agent（智能体）
```

**本质变化**：
- **过去**：追求榜单分数、模型参数量、对话能力
- **未来**：追求真实场景任务完成率、可验证结果、商业价值创造

## 核心议题详细讨论

### 一、AI范式的根本转变

#### 1. Chat时代的终结

与会者达成共识：**大模型竞争已从单纯的"聊天（Chat）"能力和榜单分数，转向能够进入真实环境、可验证并完成复杂任务的智能体（Agent）阶段。**

**唐杰观点**：
- Chat这一代的问题已"差不多解决"
- 行业重心正从"更会聊天"转向"能完成任务"
- 过去市场追逐的是模型在考卷上的"分数"，现在的核心指标变成了在真实环境中的"跑通率"

**林俊旸观点**：
- AI如果仅是更聪明的搜索框，其价值有限
- 但如果AI能将任务执行的成功率从偶然变为必然，则意味着生产力的质变
- AI正在从Chat走向Thinking、Coding与Agent

#### 2. 自主学习：下一代范式的核心

**林俊旸预测**：
- 2026年最关键的"Bet（赌注）"在于AI的主动性
- 目前的AI范式是被动的，必须由人类通过Prompt启动
- 未来范式：环境直接Prompt模型，让AI根据环境反馈自主思考、自主决策，而不仅仅是响应指令

**姚顺雨观点**：
- AGI接下来最重要的点就是自主学习
- 自主学习是一个渐变的过程，由数据和任务驱动的场景演进
- 举例：目前Claude 95%的代码已经是由Claude自己编写，这本身就是一种自我优化的循环
- ChatGPT利用用户数据风格，光标修改每隔几小时就根据用户数据进行更新，都是自主学习的早期信号

**关于AI风险的讨论**：
林俊旸表达了对AI主动性的担忧：
- "我最担心的是他做一些不该做的事。比如主动产生一个想法，往会场里扔一颗炸弹。"
- 培养AI就像培养小孩，必须注入正确的方向指引，这是主动学习范式下不可回避的命题

**关键信号**：
- Claude 95%的代码由自己编写
- ChatGPT每几小时根据用户数据更新
- 光标修改实时学习用户习惯

### 二、Scaling Law的效率革命

#### 1. 当前的困境

**唐杰指出的问题**：
- 从2025年初的10T数据，到现在的30T甚至未来的100T
- 单纯增加数据量的收益正在递减
- "如果你不创新，花掉10个亿、20个亿，收益却很小，这就不值得了"

#### 2. 新的衡量标准：智能效率（Intelligence Efficiency）

**唐杰提出**：
- 提升智能上限"最笨"的方法是Scaling
- 更聪明的做法应该是定义智能效率
- 即如何用更少的算力投入、更小的Scaling规模，获得同等甚至更高的智能提升
- 在持续学习、Memory（记忆）以及多模态领域，这种追求效率的范式变革一定会发生

#### 3. Scaling Law的内涵变化

**杨植麟观点**：
- Scaling Law还远未到达终点，但其内涵发生了变化
- Scaling不仅仅意味着堆算力，而是在架构、优化器、数据层面做技术改进
- 目的是为了让模型拥有更好的"Taste（品味）"，这正是模型之间形成护城河的关键
- Token效率与长文本将是决定AI智力水平的核心因素

**林俊旸观点**：
- 提出了"测试时扩展"（Test-time Scaling）思路
- 与其在训练阶段死磕数据量，o1系列证明了另一条路
- 即在推理时就投入更多计算资源，让模型变强
- "人类与AI不断交互，只会让它上下文变得越来越长，导致AI变得越来越笨，这件事很烦人"

**新范式**：
- 用更少算力获得同等/更高智能
- 测试时扩展（Test-time Scaling）：推理阶段投入更多算力而非训练阶段
- Token效率与长文本成为核心竞争力

**警示**：花10-20亿投入却收益很小，这种创新不值得

### 三、模型分化：To B与To C的二元分化

#### 1. To B端（企业场景）：智能即收益

**姚顺雨观察**：
- 在B端（企业）场景，智能直接等同于收益
- 美国市场用户愿意为强模型支付数倍溢价
- 因为强模型能将任务正确率从60%提升到90%，大幅降低人工监控成本
- 因此在To B领域，强弱模型的分化会越来越极端

**特点**：
- 逻辑：智能 = 收益
- 趋势：强弱模型分化极端化
- 付费意愿：强模型（60%→90%准确率）值得数倍溢价
- 关键：降低人工监控成本

#### 2. To C端（用户场景）：垂直整合路线

**姚顺雨观点**：
- To C的分化不在于模型变大，而在于Context（上下文）的厚度
- To C产品应通过垂直整合，让模型感知用户的实时环境，如天气、聊天记录等，而非盲目Scaling
- 因此To C应用必须走"垂直整合"路线，让模型与产品强耦合

**特点**：
- 逻辑：模型智能 ≠ DAU增长
- 趋势：垂直整合，模型与产品强耦合
- 关键：Context（上下文）厚度比模型大小更重要
- 瓶颈：
  1. 环境和部署问题
  2. 用户教育问题

#### 3. 林俊旸关于生态位的观点

**核心观点**：
- 千问在未来AGI的生态位与公司基因关系并不大
- 今天To B和To C都是在服务真实的人类
- 这个问题本质是，应该怎么让人类世界变得更好
- 以Anthropic为例，其在Coding（编程）领域的成功是由于频繁与B端客户交流，发现了真实场景中巨大的Token消耗
- 而在中国，Coding的Token消耗还没有那么大

#### 4. 杨强院士的补充观点

**人工智能科学家、加拿大皇家科学院院士杨强提出**：
- 当大模型进入稳态，学术界应研究"智能上限和幻觉平衡"
- 即投入多少资源能换取多少错误率的降低，这是一种类似经济学的风险收益平衡
- 建议大模型研究应借鉴人类的睡眠机制——晚上睡觉事实上是在清理噪音，使得第二天学习的准确率持续提升
- 大模型的持续学习中，同样需要类似的噪音清理

### 四、Agent时代的商业价值爆发

#### 1. 对Agent价值的期待

**市场普遍预期**：2026年将不再是"更强模型之年"，而是模型真正跑通业务流程、创造商业价值的关键年份

**姚顺雨观点**：
- To B方面，Agent正处在不断上升的曲线上，目前没有变慢的趋势
- 但在To C场景下，DAU和模型智能很多时候不仅不相关，甚至会相反
- 指出除了模型本身还有两个瓶颈：
  1. 环境和部署（Deployment）问题
  2. 教育问题
- 认为在今天的中国，想要扩大To C Agent的机会，教育大家如何更好地使用AI产品是一个最大的、最有意义的事情

#### 2. 通用Agent的创业机遇

**林俊旸观点**：
- 引用了Peak（Manus联合创始人）的一句话："通用Agent最有意思的事情在于解决长尾问题，或者说今天AI更大的魅力是在长尾"
- 较为认同"模型即产品"这个观点
- 认为Agent的未来应当不仅仅出现在电脑里，而是和真实物理世界交互——那才是Agent能做长时间工作的情景
- 预判称，一些在电脑环境里做的，今年很快就可以完成
- 但接下来三到五年要完成的Agent任务，可能和具身智能会结合起来

**应用方向**：
- 短期（1年内）：电脑环境内的任务自动化
- 中期（3-5年）：与具身智能结合，进入物理世界
- 长期：解决长尾问题

#### 3. 杨强对Agent发展的阶段划分

**四个阶段预测**：
- **初级阶段**（当前）：目标是人定义的，规划也是人做的
- **未来阶段**：大模型会观察人的工作，把人的过程数据利用起来
- **最终目标**：目标和规划都由大模型来定义，Agent应该是由大模型内生的一个系统

**Agent发展三阶段**（杨强）：
1. **当前**：人定义目标，人做规划
2. **未来**：大模型观察人类工作，利用过程数据
3. **终极**：大模型内生系统，目标和规划由AI定义

#### 4. 唐杰的务实观点

**关键决定因素**（唐杰）：
- Agent这个事情的价值、能不能真正帮助人
- 成本
- 做应用的速度
- 现在的大模型更多是拼速度、拼时间
- "也许我们代码正确了，也许我们就会在这方面走的更远一点，但也许失败以后就半年，半年就没了"

**成功要素**（唐杰）：
1. 价值：能否真正帮助人
2. 成本：是否可承受
3. 速度：应用开发速度

### 五、中美AI竞争：差距与赶超

#### 1. 对差距的清醒认识

**峰会传递的情绪**：冷静而现实

**关于"中国领先概率"的讨论**：
- 尽管市场热衷于讨论"崛起"与"占榜"，但多位核心人物将中国在下一代范式中领先的概率评估为**不超过20%**

**唐杰的观点**：
- 虽然中国模型在快速迭代，但差距可能并没有缩小
- 因为美国还有大量未开源的闭源模型

#### 2. 结构性差异

| 维度 | 美国 | 中国 |
|------|------|------|
| 算力投入方向 | 下一代研究（高风险探索） | 产品化 |
| 创业文化 | 冒险精神、前沿探索 | 跟随优化 |
| To B市场 | 付费意愿强、文化成熟 | 付费意愿待培养 |
| 优势领域 | 范式创新、原始创新 | 工程实现、效率优化 |

**中美算力投入的本质差异**：
- **美国**：倾向于投资"下一代研究"的高风险探索
- **中国**：目前的算力更多被交付与产品化占据

**三道门槛**：
1. 算力瓶颈
2. 软硬生态的完善度
3. To B市场的付费意愿

**唐杰警告**：
- 如果生态系统只奖励确定性的榜单数字，而将冒险精神挤出组织结构，那么反超将难以实现

#### 3. 中国的赶超机会

**姚顺雨观点**：
- 中国要从今天的跟随者变成未来的引领者，有几个比较关键的点：
  1. 算力突破
  2. 除了To C之外，能不能有更成熟或者更好的To B的市场
  3. 有没有机会在国际的商业环境竞争
- 坦言今天很多做生产力或者做To B的模型和应用还是会诞生在美国
- 原因是美国的支付意愿更强，文化更好
- 更重要的是，主观上有更多有创业精神或者冒险精神的人，真的想要去做前沿探索或者新的范式突破的事情

**核心判断**：
- "目前来看，一个范式一旦发生，我们可以用很少的卡、很高的效率去局部做得更好，我们到底能不能引领新的范式？这可能是今天中国唯一要解决的问题。"
- 希望大家能走出"榜单的束缚"，坚持自己觉得是不是正确的过程

**林俊旸观点**：
- 同样强调了冒险和创新的重要性

#### 4. 唐杰总结的三大机会

**在"差距"之外指出中国市场的赶超机会**：
1. 一群聪明人真的敢做特别冒险的事
2. 更好的环境，包括营商环境
3. 回到个人身上，能不能坚持的问题
   - "我们能不能愿意在一条路上敢做、敢冒险，而且环境还不错"

## 技术路线图

### Chat时代（已终结）
```
输入 → 模型生成 → 输出
↓
指标：榜单分数、对话质量
```

### Agent时代（当前转型）
```
任务理解 → 自主规划 → 工具调用 → 结果验证 → 迭代优化
↓
指标：任务完成率、商业价值、ROI
```

### 自主学习时代（未来方向）
```
环境感知 → 自主目标设定 → 自主决策 → 持续学习 → 自我进化
↓
指标：智能效率、适应性、创造性
```

## 对企业与投资者的启示

### 对AI企业的建议

1. **走出榜单束缚**
   - 不要被Benchmark分数绑架
   - 关注真实场景的"跑通率"
   - 坚持自己认为正确的方向

2. **明确赛道定位**
   - To B：深耕特定场景，提升任务准确率
   - To C：垂直整合，强化用户体验和上下文理解

3. **重视智能效率**
   - 不是盲目堆算力
   - 关注Token效率、长文本能力
   - 探索测试时扩展等新路径

4. **布局Agent生态**
   - 不要等待"更强模型"
   - 现在就开始构建Agent应用
   - 关注长尾问题解决能力

5. **培养冒险精神**
   - 鼓励前沿探索
   - 容忍失败
   - 建立创新文化

### 对投资者的建议

1. **关注效率型公司**
   - 不是算力堆砌型
   - 而是智能效率领先型

2. **看好Agent赛道**
   - 2026年是商业价值爆发年
   - 企业级Agent需求旺盛
   - 关注垂直整合能力

3. **理性看待中美差距**
   - 中国在工程化和效率上有优势
   - 但在原始创新上仍落后
   - 范式创新是关键变量

4. **警惕伪Agent概念**
   - 真正的Agent能自主规划、执行、验证
   - 不是简单的RAG或Workflow
   - 关注真实的商业价值创造
