# 文件组织规则

## 📝 文档时间戳规则

**所有创建的文档必须包含精确到秒的时间戳。**

**时间戳格式**：
```markdown
> **创建时间**: YYYY-MM-DD HH:MM:SS
```

**示例**：
- ✅ 正确：`> **创建时间**: 2026-01-28 14:30:55`
- ❌ 错误：`> **日期**: 2026-01-28`
- ❌ 错误：`> **创建时间**: 2026-01-28`

**原因**：只有日期无法分辨文档修改的先后顺序，精确到秒可以明确版本顺序。

## 文档生成位置规则

### 默认文档存放位置

除非用户**明确指定路径**或文档属于**特定项目**，否则所有生成的文档统一存放至：

```
/home/gwh/dashgo_rl_project/.claude-temp/docs/
```

### 适用范围

✅ **需要存放至 `Si Yuan\claude\` 的文档：**
- 科普类文档（如投资策略、技术概念解释等）
- 分析报告（如行业分析、政策分析等）
- 学习笔记类文档
- 任何非项目相关的独立文档

❌ **不需要存放至 `Si Yuan\claude\` 的文档：**
- 用户明确指定路径的文档
- 属于特定项目的代码文件
- 项目内的配置文件、文档等

### 文件命名规范

- 使用中文文件名（便于识别）
- 格式：`主题_类型_YYYY-MM-DD.md`
  - 例如：`低波投资策略_科普_2025-01-13.md`
  - 例如：`阿根廷现状_分析_2026-01-12.md`
- 如果日期不重要，可省略：`低波投资策略科普.md`

### 示例

**正确操作**：
```
用户：生成一个关于xxx的文档
AI：[将文档生成到] /home/gwh/dashgo_rl_project/.claude-temp/docs/xxx科普.md
```

**用户指定路径**：
```
用户：生成一个文档到 docs/tutorial.md
AI：[将文档生成到] docs/tutorial.md（按用户指定路径）
```

## 临时文件管理规则

### 临时文件存放位置

所有临时文件统一存放至：

```
/home/gwh/dashgo_rl_project/.claude-temp
```

### 临时文件类型

- `tmpclaude-*-cwd` - Claude 临时工作目录
- 临时测试文件
- 临时脚本文件
- 任何不需要长期保留的文件

### 清理策略

- 对话结束后可清理 `.claude-temp/` 内的临时文件
- 保留 `.claude-temp/` 文件夹本身
- 使用清理脚本：`cleanup-claude-temp.sh` 或 `cleanup-claude-temp.bat`

### 注意事项

⚠️ **禁止**在根目录（`/home/gwh/dashgo_rl_project/`）下生成散乱的临时文件
⚠️ **禁止**将临时文件提交到 Git 仓库（已在 `.gitignore` 中配置）

## 执行检查清单

### 生成新文档时

```
□ 是否为项目文件？
  - 是 → 存放到项目目录
  - 否 → 继续检查

□ 用户是否明确指定路径？
  - 是 → 使用用户指定路径
  - 否 → 存放到 .claude-temp/docs/

□ 检查目标目录是否存在
  - 不存在 → 创建目录

□ 使用规范的文件命名
```

### 创建临时文件时

```
□ 确认是否为临时文件
□ 存放到 .claude-temp/ 目录
□ 对话结束后提醒用户清理
```
